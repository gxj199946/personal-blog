vue3+ts+pinia+element+tailwindcss+graphql项目搭建

Posted by Puppet on Tue, Nov 29, 2022

一、创建vue3.2+ts项目

1.创建项目命令 yarn create vite

2.选择typescript

3.cd项目根目录

4.yarn 安装依赖

5.yarn serve 运行项目

6.配置@符 在vite.config.ts中 例:

1import {resolve} from 'path'
2
3export default defineConfig({
4  plugins: [vue()],
5  resolve:{
6    alias:{'@':resolve(__dirname,'src')}
7  }
8})

7.如需其他配置可访问官网 Vue.js

二、安装vue router路由

1.安装 yarn add vue-router

2.在src下创建router文件夹

3.新建index.ts文件

4.配置路由 (vue2与vue3的404配置有所不同) 例：

 1import { createRouter, createWebHashHistory, RouteRecordRaw } from "vue-router";
 2
 3
 4const routes: RouteRecordRaw[] = [
 5    {
 6        path: '/',
 7        redirect: '/login' //一级页面重定向 需在APP.vue中占位 <router-view></router-view>
 8    },
 9
10    {
11        path: '/login',
12        name: 'login',
13        component: () => import('@/views/Login.vue')
14
15
16    },
17    {
18        path: '/todo',
19        name: 'todo',
20        component: () => import('@/views/Todo.vue'),
21        children: [ //二级页面
22            {
23                path: '/todo',
24                redirect: '/todo/home' //二级页面重定向   需在一级页面.vue中占位 <router-view></router-view>
25            },
26            {
27                path: 'home',
28                component: () => import('@/views/home.vue'),
29            }
30        ]
31    },
32     {
33        path: '*',  //vue2.0 404配置
34        name: 'NotFound',
35        component: () => import('@/views/NotFound.vue')
36    },
37    {
38        path: '/:error*',  //vue3.2 404配置
39        name: 'NotFound',
40        component: () => import('@/views/NotFound.vue')
41    }
42]
43
44export const router = createRouter({
45    history: createWebHashHistory(),
46    routes
47})
48
49export default router // 导出router

5.在man.ts中引入注册 例：

1import { createApp } from 'vue'
2import router from './router/index'
3import App from './App.vue'
4
5const app= createApp(App)
6.use(router)
7.mount('#app')

6.如有其他需求配置可访问官网Vue Router

三、安装pinia小菠萝(状态管理比vuex更轻便)

1.安装命令 yarn add pinia

2.在src下新建store文件夹

3.新建index.ts文件 4.新建store.name文件 枚举

1export const enum  names{
2    TEST="TEST"
3}

4.store配置 例：

 1import { defineStore } from 'pinia'
 2import { names} from './store-name' //引入store-name文件
 3 export  const store =defineStore(names.TEST,{
 4    state: () => ({
 5        name:'4556'
 6    }),
 7    getters:{},
 8    actions:{}
 9 })
10

5.在man.ts中引入注册 例:

1import { createApp } from 'vue'
2import {createPinia} from 'pinia'
3import App from './App.vue'
4
5const app= createApp(App)
6.use(createPinia())
7.mount('#app')

6.在组件中使用,如果直接从pinia中解构数据，会丢失响应式， 使用storeToRefs可以保证解构出来的数据也是响应式的

 1    <div>
 2{{name}}
 3    </div>
 4</template>
 5
 6<script setup lang="ts">
 7import { storeToRefs } from 'pinia';
 8import {store} from '../store/index'
 9const stores=store()
10const {name,id}=storeToRefs(stores)
11</script>

7.配置完成如需查看数据可以在Google浏览器中右键点击检查 找到安装的扩展程序 vue里面就有pinia标识，如果没有安装可去下载Vue.js devtools扩展程序Vue.js devtools拖入扩展程序安装就可以正常使用了。如有其他需求可去官网,pinia 或者前往 博客 或者 B站观看视频

四、安装element-plus

1.安装命令 yarn add element-plus

2.在man.ts中引入注册 例:

1import { createApp } from 'vue'
2import ElementPlus from 'element-plus'
3import 'element-plus/dist/index.css'
4import App from './App.vue'
5
6const app= createApp(App)
7.use(ElementPlus)
8.mount('#app')

3.如有其他需求可访问官网 Element-plus

五、安装tailwindcss

1.安装命令 npm i -D vite-plugin-windicss windicss

2.然后，在你的 Vite 配置中安装插件 vite.config.js：

1import WindiCSS from 'vite-plugin-windicss'
2
3export default {
4  plugins: [
5    WindiCSS(),
6  ],
7}

3.最后，在man.ts中导入virtual:windi.css你的 Vite 条目就可以正常使用啦

1import 'virtual:windi.css'

4.如有其他配置需求可访问配置官网。样式需求可访问 样式官网

这些绑定是在考虑Vue 3@urql/vue的情况下编写的，并使用 Vue 更新的Composition API。这使 绑定功能更容易集成到您现有的功能中。@urql/vue``setup()

1.1安装@urql/vue

安装@urql/vue速度很快，无需立即安装其他软件包。

前往官网 urql

1yarn add @urql/vue graphql
2# or
3npm install --save @urql/vue graphql

大多数与 GraphQL 相关的库也需要将graphql包安装为对等依赖项，以便它们能够适应您的特定版本控制要求。这就是为什么我们需要同时graphql安装@urql/vue.

@urql/vue和包都graphql遵循语义版本控制，所有@urql/vue包都将定义一系列兼容的graphql. 但是，请注意将来的重大更改，在这种情况下，您的包管理器可能会警告您 graphql超出定义的对等依赖范围。

1.2.设置Client(可以直接看提供client)

该@urql/vue包导出了一个方法createClient，我们可以使用它来创建 GraphQL 客户端。这个中央Client管理我们所有的 GraphQL 请求和结果。在src/plugins/@urql.ts

1import { createClient } from '@urql/vue';
2
3
4const client = createClient({
5  url: 'http://localhost:3000/graphql',
6});

url当我们创建一个Client开始时，我们至少需要传递一个 API 。

另一个常见的选项是fetchOptions. 此选项允许我们自定义在将fetch请求发送到给定 API 时将传递给的选项url。我们可以传入一个选项对象或一个返回选项对象的函数。

在下面的示例中，我们将为发送到 GraphQL API的每个fetch请求添加一个令牌。Client

1const client = createClient({
2  url: 'http://localhost:3000/graphql',
3  fetchOptions: () => {
4    const token = getToken();
5    return {
6      headers: { authorization: token ? `Bearer ${token}` : '' },
7    };
8  },
9});

1.3.提供Client （第二种使用偏多）

要使用ClientVue，我们必须提供从父组件到子组件的服务。这将Client与我们的应用程序的其余部分共享一个。有@urql/vue两种不同的方法可以实现这一点。

第一种方法是使用@urql/vue’provideClient函数。这必须在您的任何父组件中调用，并Client直接接受或仅接受您传递给的选项 createClient。直接写在APP.Vue中即可使用

 1<script>
 2  import { createClient, provideClient } from '@urql/vue';
 3
 4
 5  const client = createClient({
 6    url: 'http://localhost:3000/graphql',
 7  });
 8
 9
10  provideClient(client);
11</script>

第二种方法，我们可以通过导入其默认导出并将其用作插件来使用导出的install函数并将其视为插件。@urql/vue

 1import { createApp } from 'vue';
 2import Root from './App.vue';
 3import urql from '@urql/vue';
 4
 5
 6const app = createApp(Root);
 7
 8
 9app.use(urql, {
10  url: 'http://localhost:3000/graphql',
11});
12
13
14app.mount('#app');

注意： 这一段代码写在man.ts中

1import urql from '@urql/vue';
2app.use(urql, {
3  url: 'http://localhost:3000/graphql',
4});

该插件还接受createClient’s 选项或 aClient作为其输入。
